Bexio über Supabase einrichten

Diese Anleitung beschreibt den sicheren Weg: Die App sendet die Rechnungsgrundlage an Supabase. Supabase hält den geheimen Bexio-Token und erstellt daraus Kontakt und Rechnungsentwurf in Bexio.

Warum Supabase dazwischen liegt

Die React-App läuft im Browser. Alles, was dort eingebaut wird, kann technisch ausgelesen werden. Deshalb darf der Bexio API Token nicht in die App und nicht in GitHub Pages.

Sicherer Ablauf: App → Supabase Edge Function → Bexio

Was bereits im Projekt vorbereitet ist

  • supabase/functions/bexio-oauth-start/index.ts startet die Verbindung zu Bexio.
  • supabase/functions/bexio-oauth-callback/index.ts nimmt die Bexio-Antwort entgegen.
  • supabase/functions/bexio-invoice/index.ts erstellt den Bexio-Kontakt und Rechnungsentwurf.
  • supabase/config.toml enthält die Function-Konfiguration.
  • supabase/.env.example zeigt die benötigten Secrets.
  • Die App hat im Schritt Rechnungsgrundlage den Bereich Bexio-Anbindung.

1. Supabase-Projekt erstellen

  1. Auf https://supabase.com anmelden.
  2. Neues Projekt erstellen, z.B. buechel-bestattungen.
  3. Die Project URL kopieren. Sie sieht ungefähr so aus:
https://abcxyz.supabase.co

Diese URL kommt später in der App ins Feld Supabase Project URL.

2. Bexio Developer App vorbereiten

  1. Im Bexio Developer Portal eine App erstellen.
  2. Als Redirect URL eintragen:
https://awwpowwmzlnyypsirawj.supabase.co/functions/v1/bexio-oauth-callback
  1. Client ID und Client Secret kopieren.
  2. Das Client Secret geheim halten und nicht in Screenshots oder Chats teilen.

3. Secrets in Supabase setzen

In Supabase bei Edge Functions / Secrets zuerst eintragen:

BEXIO_CLIENT_ID=deine_bexio_client_id
BEXIO_CLIENT_SECRET=dein_neues_geheimes_client_secret
BEXIO_REDIRECT_URI=https://awwpowwmzlnyypsirawj.supabase.co/functions/v1/bexio-oauth-callback
BEXIO_DEFAULT_COUNTRY_ID=1

4. Function deployen

Wenn die Supabase CLI eingerichtet ist, im Projektordner ausführen:

supabase functions deploy bexio-oauth-start
supabase functions deploy bexio-oauth-callback
supabase functions deploy bexio-invoice

Danach den Bexio-Login starten:

https://awwpowwmzlnyypsirawj.supabase.co/functions/v1/bexio-oauth-start

Nach erfolgreichem Login zeigt die Callback-Seite die Token-Secrets an. Diese Werte in Supabase als Secrets speichern:

BEXIO_ACCESS_TOKEN=...
BEXIO_REFRESH_TOKEN=...
BEXIO_TOKEN_EXPIRES_IN=...

5. App verbinden

  1. App öffnen.
  2. Todesfall öffnen.
  3. Zum Schritt Rechnungsgrundlage gehen.
  4. Im Bereich Bexio-Anbindung den Schalter Aktiv einschalten.
  5. Supabase Project URL eintragen.
  6. Bexio User ID, Konto ID, Steuer ID und Einheit ID eintragen.
  7. Auf An Bexio-Proxy senden klicken.

6. Diese Bexio-IDs brauchen wir

  • Bexio User ID: welcher Bexio-Benutzer die Rechnung erstellt.
  • Bexio Konto ID: Erlöskonto für Bestattungsdienstleistungen.
  • Bexio Steuer ID: MwSt.-Satz, z.B. 8.1%.
  • Bexio Einheit ID: Einheit für Stunden / Stück.

Diese IDs können wir beim ersten Test mit dem Bexio-Token auslesen oder aus Bexio übernehmen.
